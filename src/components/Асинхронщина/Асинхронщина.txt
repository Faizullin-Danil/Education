Основные концепции асинхронности в JavaScript, которые нужно знать, включают:

1. Callbacks (Обратные вызовы)
➖ Устаревший метод, может привести к "callback hell".
➕ Используется в старом API и в event listeners.

2. Promises (Промисы)
➖ Не отменяются после вызова.
➕ Избавляет от "callback hell", поддерживает .then(), .catch(), .finally().

3. Async/Await
➖ Работает только с промисами.
➕ Улучшает читаемость, делает код похожим на синхронный.

4. Event Loop (Цикл событий)
Механизм, который управляет асинхронным выполнением кода. Включает Call Stack, Web API, Callback Queue, Microtask Queue.
➕ Позволяет JavaScript оставаться однопоточным, но эффективным.
